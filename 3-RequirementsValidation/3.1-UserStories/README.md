```markdown
## User Stories

### Користувацька історія US1.0 – Моніторинг натягу тросів
Як система моніторингу,
я хочу здійснювати моніторинг натягу тросів у реальному часі,
щоб забезпечити безпеку працівників на майданчику.
Критерії приймального тестування:
Feature: FR1 - Моніторинг натягу тросів у реальному часі
  Scenario: Перегляд поточних показників натягу тросів
    Given система увімкнена
    When починається моніторинг натягу тросів
    Then система повинна відображати поточні показники натягу у реальному часі

### Користувацька історія US1.1 – Перегляд поточних показників
Як технічний спеціаліст,
я хочу переглядати поточні показники натягу тросів через інтерфейс,
щоб мати доступ до актуальних даних.
Критерії приймального тестування:
Feature: FR1.1 - Інтерфейс для перегляду показників
  Scenario: Перегляд поточних показників натягу
    Given система моніторингу активна
    When технічний спеціаліст відкриває інтерфейс перегляду
    Then система повинна відображати поточні показники натягу тросів

### Користувацька історія US1.2 – Зберігання історії показників
Як технічний спеціаліст,
я хочу зберігати історію натягу тросів,
щоб мати змогу аналізувати попередні показники.
Критерії приймального тестування:
Feature: FR1.2 - Зберігання історії натягу тросів
  Scenario: Збереження історії показників
    Given система здійснює моніторинг натягу
    When проходить певний проміжок часу
    Then система повинна зберегти поточні показники натягу до історії

### Користувацька історія US1.3 – Встановлення порогів натягу
Як менеджер з безпеки,
я хочу встановлювати пороги натягу для різних типів тросів,
щоб система могла попереджати про небезпечні ситуації.
Критерії приймального тестування:
Feature: FR1.3 - Встановлення порогів натягу
  Scenario: Встановлення порогу натягу для троса
    Given менеджер з безпеки увійшов у налаштування
    When він встановлює поріг натягу для конкретного типу троса
    Then система повинна зберегти новий поріг натягу

### Користувацька історія US1.4 – Сповіщення про перевищення порогів
Як менеджер з безпеки,
я хочу отримувати сповіщення про перевищення встановлених порогів натягу,
щоб оперативно реагувати на небезпечні ситуації.
Критерії приймального тестування:
Feature: FR1.4 - Сповіщення про перевищення порогів
  Scenario: Оповіщення про перевищення порогу натягу
    Given поріг натягу встановлено
    When натяг троса перевищує поріг
    Then система повинна надіслати сповіщення менеджеру з безпеки

### Користувацька історія US2.0 – Виявлення розриву троса
Як система моніторингу,
я хочу виявляти можливий розрив троса,
щоб запобігати аварійним ситуаціям на майданчику.
Критерії приймального тестування:
Feature: FR2 - Виявлення розриву троса
  Scenario: Сповіщення про можливий розрив троса
    Given система моніторить стан тросів
    When система виявляє надмірний натяг троса
    Then система повинна надіслати сповіщення працівникам та відправити звуковий сигнал

### Користувацька історія US2.1 – Автоматичний звіт про інцидент
Як менеджер з безпеки,
я хочу, щоб система автоматично створювала звіт про інцидент у разі розриву троса,
щоб мати документальне підтвердження для аналізу.
Критерії приймального тестування:
Feature: FR2.1 - Автоматичний звіт про інцидент
  Scenario: Створення звіту про інцидент
    Given система виявила можливий розрив троса
    When інцидент зареєстровано
    Then система повинна автоматично створити звіт про інцидент

### Користувацька історія US3.0 – Зберігання даних сенсорів
Як технічний спеціаліст,
я хочу зберігати дані сенсорів для подальшого аналізу,
щоб мати змогу оцінювати стан обладнання.
Критерії приймального тестування:
Feature: FR3 - Зберігання даних сенсорів
  Scenario: Збереження даних сенсорів
    Given система збирає дані з сенсорів
    When перевищення натягу троса
    Then система повинна зберегти дані у базу для подальшого аналізу

### Користувацька історія US3.1 – Архівування старих даних
Як технічний спеціаліст,
я хочу архівувати старі дані,
щоб зберігати місце на диску.
Критерії приймального тестування:
Feature: FR3.1 - Архівування старих даних
  Scenario: Архівування даних
    Given система зберігає дані
    When місце на диску обмежено
    Then система повинна автоматично архівувати старі дані
```
