## User Stories

### Користувацька історія US1.0 – Перегляд поточних показників
Як технічний спеціаліст,
я хочу переглядати поточні показники натягу тросів через інтерфейс,
щоб мати доступ до актуальних даних.
Критерії приймального тестування:
Feature: FR1.0- Інтерфейс для перегляду показників
  Scenario: Перегляд поточних показників натягу
    Given система моніторингу активна
    When технічний спеціаліст відкриває інтерфейс перегляду
    Then система повинна відображати дані поточного дня

### Користувацька історія US1.1 – Зберігання історії показників
Як технічний спеціаліст,
я хочу зберігати історію натягу тросів,
щоб мати змогу аналізувати попередні показники.
Критерії приймального тестування:
Feature: FR1.1- Зберігання історії натягу тросів
  Scenario: Збереження історії показників
    Given система здійснює моніторинг натягу
    When дані мають дату яка = поточний день -1
    Then система повинна зберегти поточні показники натягу до історії

### Користувацька історія US1.2 – Встановлення порогів натягу
Як менеджер з безпеки,
я хочу встановлювати пороги натягу для різних типів тросів,
щоб система могла попереджати про небезпечні ситуації.
Критерії приймального тестування:
Feature: FR1.2 - Встановлення порогів натягу
  Scenario: Встановлення порогу натягу для троса
    Given менеджер з безпеки увійшов у налаштування натягу тросу
    When він встановлює оптимальний поріг натягу для конкретного типу троса
    Then система зберегає новий поріг натягу
    And when він перевищений
    Then система надсилає повідомлення про можливий розрив працівнику та менеджеру з безпеки

### Користувацька історія US2.0 – Автоматичний звіт про інцидент
Як менеджер з безпеки,
я хочу, щоб система автоматично створювала звіт про інцидент у разі розриву троса,
щоб мати документальне підтвердження для аналізу.
Критерії приймального тестування:
Feature: FR2.1 - Автоматичний звіт про інцидент
  Scenario: Створення звіту про інцидент
    Given система зберігає дані
    When дані => за вказані у системі
    Then система автоматично створює звіт про інцидент

### Користувацька історія US3.0 – Архівування старих даних
Як технічний спеціаліст,
я хочу архівувати старі дані,
щоб зберігати місце на диску.
Критерії приймального тестування:
Feature: FR3.1 - Архівування старих даних
  Scenario: Архівування даних
    Given система зберігає дані
    When дані які мають дату яка = поточний день -1
    Then система автоматично архівує дані
